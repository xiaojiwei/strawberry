# Strawberry AI 流程图生成系统

> 基于原 Strawberry 流程图编辑器的 AI 增强版本
> 通过自然语言描述，一键生成专业流程图

---

## 项目简介

Strawberry AI 是在原 Strawberry 流程图编辑器基础上进行的 AI 驱动改造，实现了从自然语言描述自动生成流程图的能力。用户只需用文字描述流程，AI 即可自动生成结构化、布局合理的流程图，极大提升了流程图绘制效率。

### 核心价值

- **10倍效率提升**：从手动拖拽到自然语言生成，节省90%的时间
- **零学习成本**：无需学习复杂的绘图工具，会说话就会画图
- **灵活可控**：AI生成 + 手动微调的混合模式，兼顾效率与精确
- **多模型支持**：支持OpenAI、Claude、通义千问、豆包等主流大模型
- **增强导出**：支持PNG、PDF、JSON、Markdown等多种格式

---

## 功能特性

### AI自动生成

- **自然语言输入**：用文字描述流程，无需手动拖拽
- **智能理解**：AI理解流程逻辑，自动识别分支、循环、并行
- **自动布局**：基于层次布局算法，生成美观的流程图
- **多模型切换**：随时切换不同的AI模型

### 大模型支持

| 模型 | 提供商 | 特点 |
|------|--------|------|
| **GPT-4** | OpenAI | 理解准确，响应快速 |

| **通义千问** | 阿里云 | 国内访问快，中文优化 |

- 手动拖拽绘图
- 40+种预置图形

### 增强导出

## 快速开始

### 1. 打开应用

在浏览器中打开 `index.html`

### 2. 配置API（首次使用）

点击左侧 AI 面板的 **"配置"** 按钮，填入API配置：

**OpenAI示例：**
```
API Key: sk-proj-xxxxxxxxxxxxx
Base URL: https://api.openai.com/v1
Model: gpt-4o
```

### 3. 生成流程图

在输入框中描述流程：
```
创建一个用户登录流程：
1. 用户访问登录页
2. 输入用户名和密码
3. 点击登录按钮
4. 系统验证凭证
5. 如果验证成功，跳转到主页
6. 如果验证失败，显示错误并返回登录页
```

选择AI模型，点击 **"生成流程图"**

### 4. 调整和导出

- 手动调整节点位置
- 修改文字内容
- 导出为PNG/PDF/JSON等格式

```
└─────────────┬────────────────────────────────────┘
              ▼
┌──────────────────────────────────────────────────┐

│  ModeTool | LineTool | BaseTool | BeanXml       │
└──────────────────────────────────────────────────┘
```

### 工作流程

```
1. 用户输入自然语言描述
   ↓
2. AIFlowGenerator调用大模型API
   ↓
3. 大模型返回结构化JSON数据
   {nodes: [...], edges: [...]}
   ↓
4. AutoLayout计算节点坐标
   ↓
5. 调用ModeTool创建节点
   ↓
6. 调用LineTool创建连接线
   ↓
7. 渲染到画布
   ↓
8. 用户可手动调整
```

## 使用示例

### 示例1：用户注册流程

**描述：**
```
用户注册流程：
1. 用户填写注册表单
2. 提交注册信息
3. 验证邮箱格式
4. 验证密码强度
5. 检查用户名是否已存在
6. 如果用户名已存在，提示重新输入
7. 如果用户名可用，创建账户
```

**生成结果：**
- 11个节点（1个start、1个end、7个process、2个decision）
- 自动识别2个判断分支
- 层次布局，清晰易读

### 示例2：订单处理

**描述：**
```
电商订单处理流程：

如果库存充足：
  → 锁定库存 → 生成支付单 → 等待支付
  → 检查支付结果
  → 如果支付成功：减少库存 → 创建物流单 → 发货 → 完成
  → 如果支付失败：释放库存 → 取消订单 → 结束
如果库存不足：
  → 通知客户 → 订单关闭 → 结束
```

**生成结果：**
- 多分支结构

## 开发指南

### 扩展自定义模型

如需接入企业内部大模型，参考 `CustomAdapter.js`：

```javascript
var MyAdapter = (function() {
    function MyAdapter(config) {
        ModelAdapter.call(this, config);
    }

    MyAdapter.prototype = Object.create(ModelAdapter.prototype);

    MyAdapter.prototype.generateFlowchart = function(description) {
        // 实现你的API调用逻辑
        // 返回 {nodes: [], edges: []} 格式
    };

    return MyAdapter;
})();
```

### 自定义布局算法

可在 `AutoLayout.js` 中添加新的布局方法：
- `forceDirectedLayout()` - 力导向布局
- `gridLayout()` - 网格布局
- 自定义算法...

### 自定义提示词

修改 `PromptTemplates.js` 中的 `SYSTEM_PROMPT` 和 `EXAMPLES` 来优化生成效果。

## 故障排查

### 常见问题

**Q: 生成失败，提示API Key无效？**
A: 检查配置中的API Key是否正确，是否有足够的额度。

**Q: 生成的流程图不符合预期？**
A: 优化流程描述，提供更详细的步骤说明，或手动调整生成结果。

**Q: 导出PNG/PDF失败？**
A: 检查是否有网络问题导致第三方库加载失败，可查看浏览器控制台。

**Q: 如何使用国内大模型？**
A: 选择"通义千问"或"豆包"，配置相应的API Key即可。

### 调试方法

1. 打开浏览器开发者工具（F12）
2. 查看Console标签的错误信息
3. 查看Network标签的API请求详情
4. 检查localStorage中的配置

## 性能优化

### 生成速度

- 简单流程（5个节点）：2-3秒
- 中等流程（10-15个节点）：3-5秒
- 复杂流程（20+个节点）：5-10秒

### 优化建议

1. **描述精简**：避免过于冗长的描述
2. **分步生成**：大流程分多次生成后合并
3. **选择快速模型**：如通义千问、豆包
4. **本地缓存**：常用流程保存为JSON模板
