# Strawberry AI流程图生成系统 - 使用说明

## 项目介绍

Strawberry AI流程图生成系统是基于原有Strawberry流程图编辑器的AI增强版本，实现了通过自然语言描述自动生成流程图的功能。

### 核心特性

- **AI自动生成**：通过自然语言描述，AI自动生成流程图
- **多模型支持**：支持OpenAI、Claude、通义千问、豆包等多种大模型
- **智能布局**：自动计算节点位置，生成美观的流程图
- **手动编辑**：保留原有的所有编辑功能，可随意调整
- **增强导出**：支持PNG、PDF、JSON、Markdown等多种格式
- **配置灵活**：API配置本地保存，支持自定义模型

## 快速开始

### 1. 配置API Key

首次使用需要配置大模型的API Key：

1. 点击左侧AI面板的**"配置"**按钮
2. 选择要使用的模型（如OpenAI、Claude等）
3. 填入对应的配置信息：
   - **API Key**：从模型提供商获取的密钥
   - **Base URL**：API接口地址（通常使用默认值）
   - **Model**：模型名称（如gpt-4o、claude-3-5-sonnet-20241022等）
4. 点击**"保存"**按钮

### 2. 生成流程图

1. 在AI输入框中描述你的流程图需求，例如：
   ```
   创建一个用户注册流程，包括：
   1. 用户填写注册信息
   2. 验证邮箱格式和密码强度
   3. 检查用户名是否已存在
   4. 如果已存在，提示重新输入
   5. 如果不存在，创建账户并发送验证邮件
   6. 用户点击邮件链接激活账户
   7. 注册完成
   ```

2. 在下拉菜单中选择要使用的AI模型

3. 点击**"生成流程图"**按钮

4. 等待3-10秒，流程图将自动生成并显示在画布上

### 3. 手动调整

生成后，你可以：
- 拖动节点调整位置
- 修改节点的标题和内容
- 调整节点大小
- 添加或删除连接线
- 使用对齐工具整理布局
- 使用撤销/重做功能

### 4. 保存导出

使用工具栏的导出功能：
- **导出JSON**：保存为JSON格式，可稍后导入继续编辑
- **导出PNG**：保存为高清图片
- **导出PDF**：保存为PDF文档
- **导出XML**：使用原有格式保存（兼容旧版本）

## 流程图描述技巧

### 基本原则

1. **清晰描述流程步骤**：使用序号列出各个步骤
2. **明确分支条件**：说明"如果...则..."的判断逻辑
3. **指出循环**：说明哪些步骤需要重复执行
4. **标注起止**：明确流程的开始和结束

### 描述示例

#### 简单流程
```
用户登录流程：
1. 显示登录页面
2. 用户输入账号密码
3. 点击登录按钮
4. 验证账号密码
5. 如果正确，跳转到主页
6. 如果错误，显示错误提示并返回登录页
```

#### 复杂流程
```
订单处理流程：

开始：客户提交订单

步骤1：系统接收订单信息
步骤2：检查库存是否充足
  - 如果库存充足：继续下一步
  - 如果库存不足：通知客户并结束

步骤3：计算订单金额
步骤4：生成支付链接
步骤5：等待客户支付
步骤6：检查支付状态
  - 如果支付成功：继续下一步
  - 如果支付失败：取消订单并结束

步骤7：减少库存
步骤8：生成物流单
步骤9：发送确认邮件

结束：订单处理完成
```

#### 审批流程
```
文档审批流程：

1. 员工提交审批文档
2. 发送给部门经理
3. 部门经理审核
4. 判断：经理是否批准？
   - 批准：提交给总经理
   - 驳回：退回员工修改，返回步骤1
5. 总经理审核
6. 判断：总经理是否批准？
   - 批准：审批通过，归档文档
   - 驳回：退回员工修改，返回步骤1
7. 流程结束
```

### 关键词提示

使用这些关键词可以帮助AI更好地理解流程结构：
- **开始/结束**：明确流程边界
- **如果/否则/判断**：表示条件分支
- **循环/重复/返回**：表示循环结构
- **并行/同时**：表示并行处理
- **等待/延迟**：表示时间因素

## 节点类型说明

生成的流程图包含以下节点类型：

| 节点类型 | 说明 | 图形 | 使用场景 |
|---------|------|------|---------|
| **start** | 开始节点 | 椭圆 | 流程起点，每个流程图只有一个 |
| **end** | 结束节点 | 椭圆 | 流程终点，可以有多个 |
| **process** | 处理步骤 | 矩形 | 表示具体的操作或处理 |
| **decision** | 判断分支 | 菱形 | 表示条件判断，通常有2个出口 |
| **data** | 数据节点 | 平行四边形 | 表示数据输入或输出 |
| **document** | 文档节点 | 波浪底矩形 | 表示文档的处理 |

## 导出格式说明

### JSON格式

最完整的数据格式，包含所有节点、连接线、属性信息：
```json
{
  "metadata": {
    "version": "1.0",
    "exported": "2026-03-27T10:30:00.000Z",
    "generator": "Strawberry AI Flowchart"
  },
  "nodes": [
    {
      "id": "module10",
      "position": {"x": 100, "y": 200},
      "size": {"width": 120, "height": 60},
      "label": "开始",
      "content": "流程开始",
      "properties": {...}
    }
  ],
  "edges": [
    {
      "id": "line1",
      "from": "module10",
      "to": "module11",
      "label": "下一步",
      "type": "折线"
    }
  ]
}
```

**用途**：
- 备份流程图数据
- 导入到其他系统
- 程序化处理流程图
- 版本控制

### PNG格式

高清位图图片：
- 分辨率：2倍屏幕分辨率（高清）
- 格式：PNG（支持透明背景）
- 用途：插入PPT、文档、网页

### PDF格式

矢量文档：
- 纸张：A4
- 方向：横向（适合流程图）
- 用途：打印、分享、归档

### XML格式（原有格式）

兼容原Strawberry系统的XML格式，保留所有原有功能。

### Markdown格式

文本格式的流程图描述：
```markdown
# 流程图文档

## 节点列表
1. **开始**
   - 流程开始

2. **输入凭证**
   - 输入用户名和密码

## 流程连接
- 开始 → 输入凭证
- 输入凭证 → 验证凭证
```

## 常见问题

### Q1: 为什么生成失败？

**可能原因：**
1. API Key未配置或无效
2. 网络连接问题
3. API配额用尽
4. 描述不够清晰

**解决方法：**
- 检查配置是否正确
- 查看浏览器控制台（F12）的错误信息
- 尝试更换其他模型
- 优化流程描述

### Q2: 生成的流程图不符合预期？

**解决方法：**
1. 优化描述，提供更详细的步骤
2. 明确指出分支条件
3. 手动调整生成的流程图
4. 尝试重新生成（每次结果可能略有不同）

### Q3: 如何使用国内大模型？

国内大模型（通义千问、豆包）配置步骤：
1. 访问对应的控制台获取API Key
2. 在配置对话框中填入相关信息
3. 注意Base URL要填写正确
4. 某些模型可能需要实名认证

### Q4: API Key安全吗？

- API Key存储在浏览器的localStorage中
- 仅在本地存储，不会上传到服务器
- 建议不要在公共电脑上保存配置
- 可以随时清除配置或重置

### Q5: 生成的流程图可以编辑吗？

完全可以！生成后的流程图与手动绘制的完全一样：
- 可以移动节点位置
- 可以修改文字内容
- 可以添加或删除连接线
- 可以调整样式和属性
- 支持撤销重做

### Q6: 导出的PNG/PDF为什么有问题？

**可能原因：**
- html2canvas或jsPDF库加载失败
- 网络问题导致CDN资源加载超时

**解决方法：**
- 检查网络连接
- 打开浏览器控制台查看是否有加载错误
- 可以使用原有的XML导出作为备选

## 技术架构

### 模块组成

```
项目结构：
├── js/ai/                          # AI模块
│   ├── PromptTemplates.js          # 提示词模板
│   ├── ModelAdapter.js             # 适配器基类
│   ├── ConfigManager.js            # 配置管理
│   ├── AIFlowGenerator.js          # 核心生成器
│   └── adapters/                   # 模型适配器
│       ├── OpenAIAdapter.js
│       ├── ClaudeAdapter.js
│       ├── QwenAdapter.js
│       ├── DoubaoAdapter.js
│       └── CustomAdapter.js
├── js/layout/                      # 布局算法
│   └── AutoLayout.js               # 自动布局
├── js/export/                      # 导出功能
│   └── EnhancedExport.js           # 增强导出
└── css/
    └── ai-panel.css                # AI面板样式
```

### 工作流程

```
用户输入描述
    ↓
选择AI模型
    ↓
调用大模型API
    ↓
解析返回的JSON数据
    ↓
自动布局算法计算坐标
    ↓
渲染到画布（复用原有ModeTool和LineTool）
    ↓
用户手动调整（可选）
    ↓
导出为多种格式
```

### 自动布局算法

采用**层次布局算法**（Sugiyama算法）：
1. **拓扑排序**：检测环路并排序节点
2. **层级分配**：根据依赖关系分配节点层级
3. **交叉最小化**：优化节点顺序减少连线交叉
4. **坐标计算**：计算每个节点的最终坐标

## 最佳实践

### 1. 描述规范

- 使用中文描述（支持多语言，但中文效果最好）
- 每个步骤单独一行
- 使用序号标注步骤顺序
- 明确标注条件判断

### 2. 分步生成

对于复杂流程：
1. 先生成主流程
2. 手动调整布局
3. 再生成子流程
4. 合并到一起

### 3. 模型选择

- **快速原型**：使用通义千问或豆包（成本低）
- **高质量要求**：使用GPT-4或Claude（理解更准确）
- **自定义需求**：使用custom适配器接入企业内部模型

### 4. 配置建议

- 定期备份配置（使用导出功能）
- 为不同场景配置多个模型
- 设置合理的超时时间（默认60秒）

## 故障排查

### 问题：页面无法加载

**检查：**
1. 浏览器控制台是否有JavaScript错误
2. 所有JS文件是否正确加载
3. 第三方CDN资源是否可访问

### 问题：生成按钮无响应

**检查：**
1. 是否已配置API Key
2. 浏览器控制台是否有错误
3. 描述是否为空

### 问题：生成的节点位置混乱

**解决：**
- 这是布局算法的限制
- 可以手动调整节点位置
- 或使用对齐工具快速整理

### 问题：API调用超时

**解决：**
1. 检查网络连接
2. 简化流程描述（减少节点数量）
3. 增加超时时间设置
4. 更换其他模型
